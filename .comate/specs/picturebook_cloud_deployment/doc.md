# 绘本岛 - 云平台快速部署指南

## 📋 需求概述

将"绘本岛"（PictureBook Island）项目部署到公共网络上，使用免费的云服务，让用户可以通过互联网访问。

## 🎯 推荐方案对比

### 方案 1: Vercel + MongoDB Atlas（推荐，适合前端为主的应用）

**优点：**
- ✅ Vercel 免费额度充足（100GB 带宽/月）
- ✅ 全球 CDN 加速
- ✅ 自动 HTTPS
- ✅ 零配置部署
- ✅ MongoDB Atlas 免费版（512MB）

**缺点：**
- ❌ 后端需要部署到其他平台（Vercel 主要用于前端）
- ❌ 文件上传功能受限（Vercel Serverless Functions 有限制）

**适用场景：** 快速展示、个人项目、小型应用

---

### 方案 2: Render（推荐，全栈一体化）

**优点：**
- ✅ 前后端可部署在同一平台
- ✅ 免费版支持全栈应用
- ✅ PostgreSQL 和 MongoDB 可用
- ✅ 自动 HTTPS
- ✅ 支持持久化存储（需付费）

**缺点：**
- ❌ 免费版有休眠机制（15分钟无访问会休眠）
- ❌ 上传文件需要额外配置（需要对象存储）

**适用场景：** 全栈应用、需要后端服务

---

### 方案 3: Railway（推荐，一体化体验最佳）

**优点：**
- ✅ 一键部署前后端
- ✅ 内置 PostgreSQL/MongoDB
- ✅ 自动 HTTPS
- ✅ 支持环境变量管理
- ✅ 优秀的管理界面

**缺点：**
- ❌ 免费额度有限（$5/月起）
- ❌ 文件上传需要对象存储

**适用场景：** 快速原型开发、中小型项目

---

### 方案 4: 自建 VPS（最灵活，但需要一定技术能力）

**优点：**
- ✅ 完全控制
- ✅ 无限制
- ✅ 可部署所有功能
- ✅ 性能可控

**缺点：**
- ❌ 需要自己配置环境
- ❌ 需要自己管理安全
- ❌ 需要（少量）费用（如：$5/月的 VPS）

**适用场景：** 长期运营、生产环境

## 🚀 推荐部署方案：Render + MongoDB Atlas

综合考虑免费额度、部署难度和功能完整性，推荐使用 **Render** 部署前后端，**MongoDB Atlas** 提供数据库服务。

### 架构图

```
                    用户浏览器
                        │
                        │ HTTPS
                        ▼
              ┌─────────────────┐
              │     Render      │ (前端: Vite构建)
              │   (Static Site) │
              └────────┬────────┘
                       │
                       │ /api 请求
                       ▼
              ┌─────────────────┐
              │     Render      │ (后端: Node.js + Express)
              │  (Web Service)  │
              └────────┬────────┘
                       │
                       ▼
         ┌─────────────────────────┐
         │   MongoDB Atlas Cloud   │ (免费数据库)
         └─────────────────────────┘
```

## 📁 需要修改的文件

### 1. 前端配置调整

**文件：** `frontend/vite.config.ts`
- 修改构建输出配置
- 添加环境变量支持

**文件：** `frontend/.env.production`
- 添加生产环境 API 地址

### 2. 后端配置调整

**文件：** `server/.env.production`
- 配置 MongoDB Atlas 连接字符串
- 配置 JWT_SECRET
- 配置 CORS 源地址

**文件：** `server/package.json`
- 添加构建脚本（如需要）

### 3. 创建部署配置

**文件：** `render.yaml`（根目录）
- Render 自动部署配置

### 4. 新增功能

**文件：** `server/config/multer.js`
- 修改文件上传配置，使用内存存储或临时目录
- 注意：Render 免费版不提供持久化存储，需要配置对象存储

## 🔧 技术方案

### 数据库方案：MongoDB Atlas
- 使用 MongoDB Atlas 免费版（512MB 存储）
- 自动备份和高可用性
- 全球数据分布

### 文件存储方案（挑战）
**问题：** Render 免费版不支持持久化文件存储

**解决方案：**
1. **短期方案：** 使用内存存储 + Base64 编码（适合小文件）
2. **长期方案：** 集成云对象存储
   - Cloudinary（免费 25GB 存储）
   - AWS S3（免费 5GB）
   - 阿里云 OSS

### 部署流程

1. **准备阶段**
   - 注册 Render 和 MongoDB Atlas 账号
   - Fork 项目代码到 GitHub
   - 配置 MongoDB Atlas 集群

2. **部署数据库**
   - 在 MongoDB Atlas 创建集群
   - 配置网络访问（允许所有 IP 或特定 IP）
   - 创建数据库用户
   - 获取连接字符串

3. **部署后端**
   - 连接 GitHub 仓库到 Render
   - 配置环境变量
   - 部署并验证 API

4. **部署前端**
   - 在 Render 创建 Static Site
   - 配置环境变量（API 地址）
   - 部署并验证前端

5. **配置域名**（可选）
   - 绑定自定义域名
   - 配置 DNS

## ⚠️ 注意事项

### 免费版限制
- Render: 15分钟无访问自动休眠，冷启动需要 30-60 秒
- MongoDB Atlas: 512MB 存储
- 文件上传：需要额外配置对象存储

### 安全建议
1. 使用强密码和随机 JWT_SECRET
2. 启用 HTTPS（Render 自动提供）
3. 配置 CORS 白名单
4. 不要在代码中硬编码敏感信息
5. 定期更新依赖

### 性能优化
1. 启用 MongoDB 索引
2. 配置 CDN 加速静态资源
3. 使用压缩
4. 图片优化

## 🔍 备选方案

如果遇到文件上传问题，可以考虑：
1. 暂时禁用文件上传功能
2. 使用 Base64 编码存储图片（仅适合小图片）
3. 升级到付费方案获取持久化存储
4. 迁移到 VPS 方案完全控制服务器
